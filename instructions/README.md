# Padrões de Commits

Este documento apresenta os padrões de commits geralmente utilizados na comunidade de desenvolvimento de software. Seguir convenções claras ajuda na organização do histórico de commits e na colaboração da equipe.

## Tipos de Commits

| Tipo      | Descrição                                                                                   |
|-----------|---------------------------------------------------------------------------------------------|
| **feat**  | Adiciona um novo recurso ou funcionalidade ao projeto.                                     |
| **fix**   | Corrige um bug encontrado no código.                                                       |
| **docs**  | Atualiza a documentação do projeto, sem alterações no código.                             |
| **style** | Modifica o formato ou estilo do código (espaços, pontos e vírgulas, etc.) sem alterar a lógica. |
| **refactor** | Altera o código existente sem mudar seu comportamento, visando melhor organização ou performance. |
| **test**  | Adiciona ou modifica testes existentes.                                                    |
| **chore** | Realiza tarefas menores que não se enquadram nas outras categorias, como manutenção de configurações. |
| **build** | Atualiza o sistema de construção ou dependências, como alterações em arquivos de configuração. |

## Como Usar

Ao fazer commits, utilize os tipos descritos acima como prefixos das mensagens de commit. Isso permite que todos os membros da equipe compreendam rapidamente a natureza das alterações no código.
